# Fog Audit — 2026-06-12

Simon flagged foggy/washed-out backgrounds on the progress dashboard and asked whether they're broken (transparency) or intentional. This is the measured all-domain answer.

## Method

Two-timepoint screenshots (2.5s, 10.5s) of 20 archive pieces (top + newest per visual domain) via the GUI render route, measured in Node with the product's own `DecodedImageVisibility` decoder (mean luminance / brightness std / brightFraction / drift). Harness: `.quality/fog-diag4.mjs`; frames in `.quality/gui-audit/fog-*.png`.

Lesson re-learned: puppeteer `page.screenshot()` returns `Uint8Array`; `.toString('base64')` on it silently produces garbage — wrap in `Buffer.from(...)`. And in-browser measurement pages starve next to WebGL tabs ("Promise was collected") — measure in Node.

## Verdict by class

### 1. Hydra — genuinely broken (systemic, now fixed: forge PR #42)

All three hydra pieces measured **brightFraction 1.0** — zero dark pixels anywhere. Not style: the static reliability gate *required* `.brightness(>=0.6)` and prompts/contract instructed `0.75-0.95`, but hydra `brightness()` is **additive** (`rgb + amount`). A mandatory +0.75 white push forbids dark anchors mathematically. The instruction set was self-contradictory (demanded "deep darks" while mandating the operator that erases them).

A/B proof (same patch `hyd_70f3789c`, only the brightness arg changed):

| Variant | lum | brightF | Verdict |
|---|---|---|---|
| 0.84 (shipped) | 0.687 | 1.000 | grey fog |
| 0.35 | 0.196 | 0.009 | full value structure |
| none | 0.003 | 0.000 | near-black (gate's anti-black purpose is real) |

Fix: instruct 0.2–0.45, gate accepts 0.1–0.55 + fog rejection above 0.55, mirrored in both contract tiers. The three fog pieces rescored via `rescoreEntry` with provenance: hyd_70f3789c 0.78→0.50, hyd_bc3d1255 0.72→0.42, hyd_3a10f199 0.68→0.55. Residual leak (multiple `brightness()` calls stack) deliberately left to rendered-evidence scoring, which graded a live bright piece an honest 0.62 (< 0.65 admission floor).

### 2. Slate-grey flat backgrounds — craft weakness, prompt-sourced (NOT broken)

`thr_d6022aa7` (three top, 0.82: gears on flat slate — prompt literally says "on a slate sky"), `kin_c00f7cca` (kinetic top, 0.85: text on flat blue-grey), `gls_e91c2b48` (0.82: bloom rings on grey-blue). The fog read comes from flat mid-tone backdrops with no value range — and the idea pool's muted-palette vocabulary ("pewter", "slate", "dusk", "paper") invites it. This is the theme-drama lever territory: the fix is dramatic compositional themes + LIGHT & VALUE craft pressure, not a render fix.

### 3. Intentional atmosphere — working as designed

p5 abyss pieces (lum 0.06–0.16, real dark anchors), all three SVGs (lum 0.09–0.12, std ~60 — high contrast), ascii/textgen (dark, healthy), kin_f185030a, gls_77703a7b/eb358ad7. The soft glow on dark pieces is bloom doing its job.

Cosmetic note: the SVG render route shows dark art inside the white hug-content wrapper card (PR #659 design) — reads as a white halo on dark dashboards but is presentation, not artwork damage.

### Drift

No piece showed temporal accumulation in the 2.5s→10.5s window (max |drift| 0.07). The fog is steady-state, not trail buildup.
