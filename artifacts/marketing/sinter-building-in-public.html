Core Narrative: The Honest Loop

Most founders "build in public" by talking about what they will build. You have thousands of tests with real pass/fail data, a landing page that says "no cherry-picking," and a meta-harness that learns from failures.

Your narrative is not "look at my perfect product." It's "watch a creative coding agent learn to stop breaking."

The Arc

WOW → WAIT → WHY → HOW → WHAT'S NEXT

PhaseEmotionContent
WOWAwe15-60s clips of generated art/music
WAITSkepticism"But does it actually work?" — dogfood dashboard
WHYCuriosityThe meta-harness: why it fails, how it learns
HOWUnderstandingTechnical deep-dives, model comparison
WHAT'S NEXTAnticipationRoadmap, community challenges, live gen

Never skip WAIT. The failures are your credibility. The landing page says 0.77 average — lean into that.

Content Pillars

Script 3: "The Hall of Shame"

Purpose: Build trust by showing failures. Length: 60-90s

[0:00-0:05] Hook "Everyone shows AI successes. Let me show you what failure looks like when you're generating creative code at scale." [0:05-0:15] The Data Show dogfood dashboard. "2,496 tests. 1,847 passed. 649 failed. That's a 74% pass rate." [0:15-0:25] Failure 1: Remotion Show 0/3 Remotion results. "Remotion video: total failure. The model generated React components, but none of them were valid Remotion compositions. Just... broken JSX." [0:25-0:35] Failure 2: ASCII Art Show ASCII output with invalid characters. "ASCII art: the model kept using Unicode box-drawing characters. That's not ASCII. The validator caught it." [0:35-0:45] Failure 3: Three.js Show Three.js scene that's just a gray cube. "Three.js 'crystal cave' prompt: the model forgot to add lights. Just a black screen with invisible geometry. Classic." [0:45-0:60] The Fix "But here's the thing — the meta-harness logs every failure with metadata. After 50 similar Three.js lighting failures, it injected 'add ambient and directional lights' into the prompt template. Next batch: 89% pass rate." [0:60-0:75] The Philosophy "I could hide the failures. Show only the 0.89-score jellyfish. But then you'd never trust the score. Real outputs. Honest scores. No cherry-picking. That's the only way this works." [0:75-0:90] CTA "What domain should I stress-test next? Reply with your worst nightmare prompt."

Script 6: Live Generation Stream

Format: Twitter Spaces / Twitch / YouTube Live | Duration: 30-60 min

Setup: - Screen share: terminal + browser side by side - Chat visible - Background audio: ambient Strudel loop Flow: 1. Intro (5 min): "No editing. If it fails, you see it fail." 2. Prompt 1 (10 min): Pick from chat. Generate. Show score. Discuss. 3. Prompt 2 (10 min): Harder prompt. Maybe it fails. Debug live. 4. The Comparison (15 min): Same prompt, 2-3 models. Compare live. 5. Q&A (15 min): Answer technical questions. Show code if asked. 6. Outro (5 min): "All outputs added to gallery with your username." Pro tip: When something fails, that's the best content. Don't rush to fix it. Explain WHY it failed, check the harness logs, show the pattern detection.

What Makes This Work

1. Honesty is differentiation. Everyone else cherry-picks. You literally built "no cherry-picking" into the product.

2. Data is content. The dogfood pipeline generates infinite storytelling material.

3. Visuals are shareable. p5.js and GLSL outputs are inherently beautiful.

4. Failure is engaging. Watching something break, then watching the system learn why, is more compelling than perfection.

5. You're both roles. The creative coder AND the engineer who built the evaluation pipeline. That duality is rare.
